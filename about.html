Inspired by Nicolle Sullivan’s advice to “grep your styles”, CSS Dig unearths CSS properties and values from almost any website to help refactor, standardize and maintain.

To start digging, enter the URL of any website and select or enter any additional properties you want to search. Here are a few example CSS Digs:

  • apple.com
  • yahoo.com
  • aol.com (lots and lots of pretty blues)
  • msn.com (note the lone z-index value of 100000000)

How can it help me?

Looking through the CSS Dig results of your own websites you might notice you have too many colors, inconsistent font sizing, and strange z-index values. This knowledge can be the basis for a number of actions:

  • A site-wide refactor. Set better default margins and padding, consoldate colors, font sizes and line-heights.
  • Establish code syntax standards. #CCC and #cccccc are the same color but should they both appear? Provide your developers with coding guidelines.
  • Ensure that your style-guide standards are being followed, preventing rouge designers and developers from junking up the CSS.

How does it work?

CSS Dig collects the CSS from all link tags with rel=stylesheet in the HTML at the URL you provide, excluding any that for whatever reason cannot be reached (paid web fonts, for example). It then searches through that combined code, grouping any matches with the properties provided in the form. Note that it will return results inside CSS comments.

What doesn't it do?

Since CSS Dig looks exclusively for CSS in link tags there are some limitations.

  • It doesn't parse any properties between style tags in the head.
  • It also doesn't parse inline styles.
  • It may have trouble with redirects and fail silently on some websites (I'm working on that).
